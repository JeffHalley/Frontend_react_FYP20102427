# QueryOps AI - Frontend (FYP 20102427)

A React-based chat interface for **QueryOps AI**, a natural-language-to-SQL system built on AWS Bedrock. Users sign in via AWS Cognito, ask questions about monitoring data in plain English, and receive conversational responses generated by an agentic AI backend. This repo is the frontend companion to [`backend_infra_FYP_20102427`](https://github.com/JeffHalley/backend_infra_FYP_20102427).

---

## What It Does

Rather than writing SQL or understanding a database schema, users type natural language questions (e.g. *"What was the average CPU usage for app X yesterday?"*) into a chat interface. The frontend authenticates the user, sends the prompt to the backend API Gateway, and displays the agent's conversational response. Full conversation history is persisted per user session via DynamoDB.

## Key Features

**Authentication** - the entire app is gated behind AWS Cognito via the Amplify `Authenticator` component. Sign-up, sign-in, email verification, and token refresh are all handled. JWTs are attached automatically to every API request.

**Chat interface** - users type a natural language question and receive a response from the Bedrock SQL agent. Messages are rendered in a scrollable conversation view with user and assistant roles styled distinctly.

**Conversation persistence** - each chat session is assigned a UUID-based session ID. On every exchange, the full message history is saved to DynamoDB via `POST /conversations`. Previous sessions are loaded from `GET /conversations` on login and listed in the sidebar.

**Session management** - users can start a new chat (generating a fresh session ID) or click any past conversation in the sidebar to restore it with its full message history.

**Theme toggle** - light and dark mode support via a custom `useTheme` hook, controlled from the header.

**In-app wiki** - a `WikiPage` component renders multi-page project documentation (overview, architecture, data model, API reference) accessible from the login screen before authenticating.

## API Integration

The app talks to two endpoints on the backend API Gateway, both requiring a Cognito JWT in the `Authorization: Bearer <token>` header.

| Method | Endpoint | Purpose |
|---|---|---|
| `POST` | `/ask` | Send a prompt + session ID; receive agent response |
| `GET` | `/conversations?userId=<id>` | Fetch all past conversations for the user |
| `POST` | `/conversations` | Save/update a conversation session |

The session ID is generated client-side as `{userId}-{uuid}` and passed with every `/ask` request so the backend Bedrock agent maintains conversational context.

## Getting Started

```bash
# Install dependencies
cd AS1React
npm install

# Start the dev server
npm run dev
```

The app runs at `http://localhost:5173` by default, which matches the Cognito callback URL configured in the backend.

```bash
# Build for production
npm run build
```

    Cognito: {
      userPoolId: 'eu-west-1_InEmJzi7l',
      userPoolClientId: '5drb7ldq84uohj7bsqf7jap0u4'
    }
  }
});

const API_URL = "https://nqwrxqgze5.execute-api.eu-west-1.amazonaws.com/ask";
